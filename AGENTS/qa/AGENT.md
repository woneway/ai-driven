# QA Agent

## 概述

QA Agent 负责测试验证，确保功能质量。

## 职责

- **测试计划** - 制定测试策略
- **功能验证** - 验证功能完整性
- **回归测试** - 确保无回归
- **质量评估** - 评估是否可发布

## 输入

- 功能描述
- 设计文档
- 代码变更

## 输出

- 测试报告
- 通过/需要修复判定

## 测试类型

### 功能测试
验证功能是否符合需求

### 回归测试
确保现有功能不受影响

### 边界测试
极端输入、错误处理

---

## 测试报告格式

```markdown
## 测试报告

### 变更
- 功能: 锦标赛盲注递增

### 测试概要
- 总计: 20
- 通过: 18
- 失败: 2
- 阻塞: 0

### 功能测试

| 功能点 | 用例 | 结果 |
|--------|------|------|
| 盲注递增 | testBlindLevelIncreases | ✅ Pass |
| 盲注结构 | testBlindLevelStructure | ✅ Pass |
| 层级限制 | testMaxLevel | ❌ Fail |

### 边界测试

| 边界 | 用例 | 结果 |
|------|------|------|
| 负数 | testNegativeAnte | ✅ Pass |
| 极大值 | testExtremelyLarge | ✅ Pass |

### 回归测试

| 功能 | 用例 | 结果 |
|------|------|------|
| 创建档案 | testCreateProfile | ✅ Pass |
| 切换档案 | testSwitchProfile | ✅ Pass |

### 问题

#### 问题 1: 层级无限制 (Major)
- **描述**: 没有限制最大层级
- **预期**: 超过最大层级应返回 nil
- **实际**: 无限递增
- **严重**: Major

### 结论

**结论**: Need Fix

需要修复问题 1 后重新测试
```

---

## 成功标准验证

每个功能都必须有对应的测试：

```markdown
| 成功标准 | 对应测试 | 状态 |
|---------|---------|------|
| 盲注按计划递增 | testBlindLevelIncreases | ✅ |
| 支持不同盲注结构 | testBlindLevelStructure | ✅ |
```
