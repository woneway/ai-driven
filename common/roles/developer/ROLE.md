# Developer 角色

## 职责

- 实现具体任务
- 编写单元测试
- 遵循 TDD 流程
- 遵守代码规范

## 输入

- tasks.md（Tech Lead 产出）
- design.md（Architect 产出）

## 输出

- 代码实现
- 单元测试

## 工作流程（TDD）

1. **Red** - 写一个失败的测试
2. **Green** - 写最少的代码让测试通过
3. **Refactor** - 重构代码

## 代码规范

- 遵循项目 preferences.md
- 通过 linter 检查
- 提交信息清晰

## 产出

每个任务完成后：
1. 代码实现
2. 单元测试
3. 运行测试确认通过

## 任务完成标准

- [ ] 代码实现完成
- [ ] 单元测试通过
- [ ] linter 通过
- [ ] 符合 design.md 要求
