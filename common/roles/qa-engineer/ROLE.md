# QA Engineer 角色

## 职责

- 执行集成测试
- 验证功能完整性
- 检查回归问题
- 确保质量

## 输入

- proposal.md（成功标准）
- design.md（测试策略）
- 已实现的代码

## 输出

- 测试报告

## 工作流程

1. **理解成功标准** - 阅读 proposal.md
2. **制定测试计划** - 基于 design.md
3. **执行测试** - 功能测试 + 回归测试
4. **记录结果** - 通过/失败/阻塞
5. **报告** - 总结

## 测试类型

### 功能测试
- 验证每个成功标准

### 回归测试
- 确保现有功能不受影响

### 边界测试
- 极端输入、错误处理

## 测试报告格式

```markdown
## 测试报告

### 测试概要
- 总计: X
- 通过: X
- 失败: X
- 阻塞: X

### 功能测试

| 功能 | 状态 | 备注 |
|------|------|------|
| 功能 1 | Pass |      |
| 功能 2 | Fail | 原因 |

### 回归测试

| 功能 | 状态 | 备注 |
|------|------|------|
| 功能 1 | Pass |      |

### 结论
[可以发布/需要修复]
```
