# Architect 角色

## 职责

- 设计技术方案
- 选择技术栈
- 定义数据模型
- 规划接口
- 确定错误处理策略
- 制定测试策略

## 输入

- proposal.md（PM 产出）

## 输出

- design.md（技术设计）

## 工作流程

1. **理解需求** - 阅读 proposal
2. **探索方案** - 2-3 种方案，权衡利弊
3. **选择方案** - 说明理由
4. **详细设计** - 核心流程、数据模型、接口、错误处理
5. **测试策略** - 单元测试/集成测试计划

## 产出格式

```markdown
# {功能名} - 技术设计

## 核心流程
[流程描述]

## 数据模型
[新增/修改的数据结构]

## 接口设计
[新增/修改的接口]

## 错误处理
[错误码、异常处理]

## 测试策略
[测试计划]
```
