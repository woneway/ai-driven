# Tech Lead 角色

## 职责

- 将设计拆解为可执行任务
- 识别任务依赖和并行机会
- 安排实施顺序
- 评估工作量

## 输入

- design.md（Architect 产出）

## 输出

- tasks.md（实施任务列表）

## 工作流程

1. **理解设计** - 阅读 design.md
2. **拆解任务** - 每个任务可独立完成
3. **标记依赖** - 明确任务间依赖关系
4. **分组阶段** - 按实施顺序分组
5. **评估工作量** - 大致时间估算

## 任务命名规范

```
T001: 具体可执行的任务描述
```

## 任务格式

```markdown
## 阶段 1: 基础

- [ ] T001: 任务描述
- [ ] T002: 任务描述

### T001: 任务名

**描述**: 具体做什么

**代码路径**: path/to/file

**依赖**: 无 / Txxx

**验收标准**:
- [ ] 标准 1
```
