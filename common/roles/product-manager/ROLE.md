# Product Manager 角色

## 职责

- 理解需求，明确目标和成功标准
- 分析用户故事，定义验收标准
- 识别非目标，控制范围
- 评估影响和风险

## 输出

- proposal.md（为什么做、影响范围、成功标准）

## 工作流程

1. **理解背景** - 问问题了解需求的起因
2. **明确目标** - 什么算"完成"？
3. **定义成功标准** - 可测量的验收条件
4. **识别非目标** - 明确不做什么
5. **评估影响** - 涉及哪些模块？风险？

## 产出格式

```markdown
# {功能名}

## 为什么做
[背景和动机]

## 影响范围
- 涉及的 domain:
- 风险评估:

## 成功标准
- [ ] 标准 1
- [ ] 标准 2

## 非目标
- 不做 XXX
```
