# Debugging Skill

## 概述

系统化调试技能。

## 何时使用

- 代码出现 Bug
- 测试失败
- 程序崩溃

## 调试流程

### 1. 重现问题

首先确认问题可以重现：
- 找到触发条件
- 记录错误信息

### 2. 定位问题

缩小问题范围：
- 是哪个模块？
- 是哪个函数？
- 是哪一行？

### 3. 分析根因

理解为什么出错：
- 数据问题？
- 逻辑问题？
- 边界问题？

### 4. 修复

根据根因修复：
- 修复代码
- 添加测试防止回归

### 5. 验证

确认修复有效：
- 运行测试
- 验证边界情况

## 调试技巧

### 日志

```swift
print("[DEBUG] entering function with: \(param)")
print("[DEBUG] current state: \(state)")
```

### 断点

在关键位置设置断点：
- 入口
- 分支
- 异常

### 小黄鸭调试

向他人（或自己）解释代码：
- 逐行解释
- 找出逻辑漏洞

## 常见问题类型

| 类型 | 特征 | 解决方法 |
|------|------|---------|
| 空值 | nil/undefined | 添加 guard/if let |
| 逻辑 | 结果不符合预期 | 检查条件/运算符 |
| 边界 | 极端值出错 | 检查边界条件 |
| 并发 | 间歇性失败 | 检查竞态条件 |
| 性能 | 越来越慢 | 检查循环/内存 |

## 原则

- **不要猜测**
- **小步验证**
- **记录发现**
- **修复后验证**
