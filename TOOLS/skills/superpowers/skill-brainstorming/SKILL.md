# Brainstorming Skill

## 概述

帮助将想法转化为完整的设计方案。

## 何时使用

- 人类提出一个新需求
- 需要讨论技术方案
- 需要分析问题

## 执行步骤

### 1. 理解背景

首先了解项目上下文：
- 查看项目结构
- 查看相关文档

### 2. 提问

一个一个地提问，理解需求：
- 目标是什么？
- 有什么约束？
- 成功的标准是什么？

### 3. 探索方案

提出 2-3 种方案：
- 方案 A：优点/缺点
- 方案 B：优点/缺点

给出推荐和理由。

### 4. 确认方案

用小段落展示设计（200-300 字），确认后再继续。

## 输出

创建 `.changes/{date}_{slug}/design.md`

## 原则

- **一次只问一个问题**
- **多问选择题**
- **YAGNI** - 去掉不必要的功能
- **增量确认** - 分段确认设计
