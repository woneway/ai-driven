# 框架核心概念

本文档定义 ai-driven 框架的四大核心概念。

## Workspace

**定义**：项目工作空间，包含配置、代码、变更记录。

每个 Workspace 是独立的开发环境，通过以下方式管理：

- 配置文件：`.workspace.env`
- 项目配置：`.cursor/`
- 变更记录：`openspec/`

**用途**：隔离不同项目的配置和变更，避免相互干扰。

## /team

**定义**：唯一开发入口，一句话触发全自动开发。

用户只需描述需求（如"做一个用户登录功能"），AI 自动完成：
- 需求分析
- 代码实现
- 测试编写
- 代码审查

**触发方式**：在 Workspace 中使用 `/team` 命令。

## OpenSpec

**定义**：变更记录系统，规范每个功能的提案、设计、任务、验证。

**变更流程**：
1. **Proposal** - 描述问题和解决方案
2. **Design** - 详细设计和技术方案
3. **Tasks** - 拆分任务清单
4. **Implementation** - 代码实现
5. **Verification** - 验证实现是否符合设计
6. **Archive** - 归档变更

**用途**：确保每个变更都有完整的文档记录，便于追溯和审查。

## global_cursor

**定义**：全局能力目录，通过 symlink 挂载到 `~/.cursor/`。

**包含内容**：
- Sub-Agents - 13 个专业子代理
- Commands - 43 个命令
- Rules - 28 个编码规范
- Skills - 49 个技能包

**使用方式**：
- 各 Workspace 按需从 global_cursor 同步资源
- 保持框架资源集中管理，避免重复
